对象和集合的初始化器

对象的初始化器提供了一种简洁的语法来创建和初始化一个对象
集合的初始化器提供了一种简洁的语法来创建和填充集合。

  同学们还记得数组的使用方法吗？ 

int[] arr1 = new int[3];
arr1[0] = 10;
arr1[1] = 20;
arr1[2] = 30;
int[] arr2 = new int[] { 10, 20, 30 };

对象初始化器

有如下的一个类：

    class Student
    {
        public int Id { get; set; }
        public string Name { get; set; }
        public DateTime Birthday { get; set; }
    }

用普通方法创建和初始化一个对象

    Student stu = new Student();
    stu.Id = 1;
    stu.Name = "张三";
    stu.Birthday = Convert.ToDateTime("1996-1-1");

用对象初始化器创建和初始化一个对象

    Student stu = new Student { Id = 1, Name = "张三", 
        Birthday = Convert.ToDateTime("1996-1-1") };

  对象初始化器允许在程序中通过声明的方式直接给对象属性进行数值的初始化，而不必刻意建立有参数的构造函数
对象初始化器与构造方法的异同点
 相同点:都可以完成对象属性的初始化
 不同点:

  1. 构造函数具有强制性，而对象初始化器没有强制性

  2. 前者只能完成属性初始化，而构造函数可以完成任何需要的初始化
    工作，如对象创建

      对象初始化器时在创建对象的时候使用，而构造方法是写在类里面

  注意：使用初始化器时，要求相关的类中必须要有一个无参构造函数★★★★★

集合初始化器

用上面的类来创建和填充一个具有三个对象的泛型集合
普通方法

    Student stu1 = new Student();
    stu1.Id = 1;
    stu1.Name = "张三";
    stu1.Birthday = Convert.ToDateTime("1996-1-1");

    Student stu2 = new Student();
    stu2.Id = 2;
    stu2.Name = "李四";
    stu2.Birthday = Convert.ToDateTime("1996-1-2");

    Student stu3 = new Student();
    stu3.Id = 3;
    stu3.Name = "王五";
    stu3.Birthday = Convert.ToDateTime("1996-1-3");

    List<Student> list = new List<Student>();
    list.Add(stu1);
    list.Add(stu2); 
    list.Add(stu3);

集合初始化器

List<Student> list = new List<Student>
{
    new Student{Id = 1, Name = "张三", Birthday = Convert.ToDateTime("1996-1-1")},
    new Student{Id = 2, Name = "李四", Birthday = Convert.ToDateTime("1996-1-2")},
    new Student{Id = 3, Name = "王五", Birthday = Convert.ToDateTime("1996-1-3")},
};

注意：用初始化器，必须确保相关的类有一个公有的无参构造方法如果你在相关类中写了有参构造方法而将它的默认无参构造方法覆盖了，并且没有提供一个新的无参构造方法，那么使用对象初始化器编译的时候是不会通过的

嵌套的初始化器

考虑如下的两个类：

    class Address
    {
        public string Province { get; set; }
        public string City { get; set; }
        public string County { get; set; }
        public string Street { get; set; }
    }

    class Student
    {
        public int Id { get; set; }
        public string Name { get; set; }
        public DateTime Birthday { get; set; }
        public Address Address { get; set; }
    }

    请注意Student类中的Address属性的数据类型

    用对象初始化器初始化一个学员对象的代码如下：

    Student stu = new Student
    {
        Id = 1,
        Name = "张三",
        Birthday = Convert.ToDateTime("1996-1-1"),
        Address = new Address
        {
            Province = "湖南省",
            City = "长沙市",
            County = "雨花区",
            Street = "香樟路601号"
        }
    };

类型推断及匿名类型

一、类型推断
思考：在JavaScript中如何定义一个变量？

var i = 5;
var s = "abc";
var b = true;
var now = new Date();
var arr = new Array();

i = "欧柏泰克软件学院";
s = 123;

JavaScript属于弱类型化语言，而C#属于强类型化语言。

C#中的关键字var，可以像JavaScrpit一样的方式来声明变量，比如：

    var i = 5;
    var s = "abc";

    Console.WriteLine(i);
    Console.WriteLine(s);

  var关键字用来隐式地声明一个数据类型，var不是一种新的数据类型，变量类型是在编译期确定的，而不是在运行时确定的，它是依赖于编译器根据所赋的值来自动确定变量的类型；并且一个变量一旦赋值，在后续的代码中就不能更改其数据类型。
  如果编译器无法确定用var声明的变量类型，那代码就无法编译，因此，在用var声明变量时，必须同时初始化该变量，因为如果没有初始化值，编译器就不能确定变量的类型，比如下面的代码就无法编译：

var i;

  还可以用var关键字来声明并初始化数组：

var arr = new[] { 10, 20, 30 };

  注意：
    对于一些名称比较长的类，类实例化的时候，可以用var关键字声明的变量接收

  1. var 类型会被编译器根据初始值的类型推断出具体的类型;
  2. var 类型的局部变量必须赋予初始值
  3. 无法将null赋值给隐式类型的局部变量;
  4. var 类型变量的定义必须是在方法中或者在属性get、set 访问器中；
  5. var 不能做方法的参数；
  6. 不能当作返回值类型；
  7. 语句中只能声明一次变量，声明后不能更改类型；

二、匿名类型
在编写数据库应用程序中，我们需要编写实体类来对应数据库中的数据表，一个实体类对应于一个数据表。这些实体类往往只有一系列的属性，一个属性对应一个字段。比如：

    class Address
    {
        public string Province { get; set; }
        public string City { get; set; }
        public string County { get; set; }
        public string Street { get; set; }
    }

这样的实体类仅仅用于存储结构化的数据，一个实例对于于数据表中的一行。编写这样的类往往比较单调。我们可以匿名类型来简化这一编程模型。其原理是使用C#编译器根据要存储的数据自动创建类型，而不是定义简单的数据存储类型。
  可按如下的方式来实例化前面的Address类型：

    Address address = new Address 
    { 
        Province = "湖南省",
        City = "长沙市",
        County = "岳麓区",
        Street = "桐梓坡路579号"
    };

也可以使用匿名类型：

    var address = new 
    { 
        Province = "湖南省",
        City = "长沙市",
        County = "雨花区",
        Street = "桐梓坡路579号"
    };

区别有两点：第一，使用了var关键字；第二，在new的关键字后面没有指定类型名称，这是编译器确定我们要使用匿名类型的节奏。
在IDE中，可以智能感知出匿名类型的成员：

    Console.WriteLine("省：{0}", address.Province);
    Console.WriteLine("市：{0}", address.City);
    Console.WriteLine("县：{0}", address.County);
    Console.WriteLine("街道：{0}", address.Street);

  看得出来，Province、City、County、Street表现出来的是属性成员。
  注意：
通过匿名类创建的属性属于只读属性。这意味着要修改属性的值，就不能用匿名类型！
